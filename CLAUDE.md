# CLAUDE.md

## Project

`tarantool-migrator` is a Go library for running schema/data migrations against a Tarantool database via `github.com/tarantool/go-tarantool/v3`. Package name: `tarantool_migrator`.

## Commands

```bash
go build ./...                # compile
go test ./...                 # unit tests (no external deps needed)
go test -coverprofile=coverage.txt -covermode=atomic ./...
golangci-lint run             # lint (must be 0 issues before committing)
```

Regenerate the pool mock after changing `mocks/gen.go`:
```bash
cd mocks && go generate
```

## Architecture

| File                       | Responsibility                                                                                  |
|----------------------------|-------------------------------------------------------------------------------------------------|
| `migrator.go`              | Public API — `NewMigrator`, `Migrate`, `RollbackLast`, option funcs                             |
| `executor.go`              | Internal — `executor` interface, `executorBase` + `newExecutor` factory, common Tarantool calls |
| `executor_notx.go`         | `noTxExecutor` — `applyMigration`, `rollbackMigration` without transactions                     |
| `migration.go`             | `Migration` struct, `MigrateFunc`/`RollbackFunc` types, `NewGenericMigrateFunction`             |
| `migrations_collection.go` | `MigrationsCollection` slice — sorted by ID, `Find`                                             |
| `migration_file.go`        | Parses `<name>.<up                                                                              |down>.lua` filenames |
| `loader_embed.go`          | `EmbedFsLoader` — loads `.lua` files from an `embed.FS`                                         |
| `lua_embed.go`             | Embeds `lua/migrations/create_migrations_space.up.lua`                                          |
| `options.go`               | `Options` struct + `DefaultOptions`                                                             |
| `errors.go`                | Sentinel errors                                                                                 |
| `logger.go`                | `DefaultLogger`, `DebugLogger`, `SilentLogger`                                                  |
| `mocks/pool.go`            | `PoolerMock` — generated by `moq`, used in tests                                                |
| `helpers_test.go`          | `newMigrationTupleStubResponseBody()` shared test helper                                        |
| `tmp/`                     | Local testdrive binary (not part of the library, excluded from lint)                            |

## Key patterns

**Pool usage** — always pass `pool.ModeAny` for reads, `pool.ModeRW` for writes. These come from `opts.ReadMode` / `opts.WriteMode`.

**Migrations space** — the space name is set in `Options.MigrationsSpace` (default `"migrations"`). The Lua bootstrap script uses the `_migrations_space_` token, replaced at runtime via `strings.ReplaceAll`.

**Migration files** — naming convention: `<id>.<up|down>.lua`. Files prefixed with `--` are skipped. IDs are sorted lexicographically, so timestamp-based IDs like `202410082345_name` work naturally.

## Testing

Tests use `testify/suite` for `Executor` tests and table-driven tests elsewhere. No integration tests — all Tarantool calls are mocked.

Mock setup per test (never in `SetupTest`):
```go
mockDoer := test_helpers.NewMockDoer(suite.T())
mockDoer.AddResponseRaw([][]interface{}{})           // success, no data
mockDoer.AddResponseRaw(newMigrationTupleStubResponseBody()) // migration tuple
mockDoer.AddResponseError(fmt.Errorf("some error")) // error
suite.mock.DoFunc = func(req tarantool.Request, mode pool.Mode) tarantool.Future {
    return mockDoer.Do(req)
}
```

Responses are consumed in order.

Request assertions use `reflect.ValueOf` to read unexported fields (see existing tests for patterns).

## Linting rules

Active linters: `errcheck`, `funlen` (70 lines / 50 statements), `gocognit`, `goconst`, `gocritic`, `gocyclo`, `gosec`, `govet`, `ineffassign`, `lll`, `misspell`, `nakedret`, `nlreturn`, `prealloc`, `staticcheck`, `unconvert`, `unused`, `wsl_v5`.

**`nlreturn`** — blank line required immediately before every `return`.

**`wsl_v5`** — blank line required between statement groups (assignments, calls, if-blocks).

`funlen`, `goconst`, `lll`, `staticcheck`, `wsl_v5` are excluded for `*_test.go` files. The `lua/`, `mocks/`, and `tmp/` directories are excluded entirely.

## Local Tarantool dev

```bash
docker compose up -d          # tarantool/tarantool:3.7 on 127.0.0.1:3301
cd tmp && go run .            # run testdrive against live instance
docker compose down           # stop (data persists)
docker compose down -v        # stop and wipe data
```

Credentials: user `migrator`, password `migrator-pwd` (see `.docker/config.yaml`).
